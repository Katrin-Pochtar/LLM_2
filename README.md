# Fine-Tuning LLM для стилизации персонажа: Кот Бегемот

## Структура репозитория

| Файл | Описание |
|---|---|
| `prepare_dataset_behemot_gpt.ipynb` | Подготовка датасета |
| `main.ipynb` | Обучение, оценка, инференс |

## Данные

**Источник**: текст романа «Мастер и Маргарита» (756 048 символов).

**Сбор данных** (см. `prepare_dataset_behemot_gpt.ipynb`):
1. Текст разбит на 290 чанков (~3000 символов, overlap 200).
2. GPT-4o извлёк 107 уникальных реплик Бегемота из оригинального текста.
3. GPT-4o сгенерировал вопросы к каждой реплике - 107 пар «вопрос - цитата из книги». Вопросы сгенерированы, ответы из книги.
4. GPT-4o дополнительно сгенерировал 60 пар на разнообразные темы (еда, философия, технологии, любовь и т.д.) в стиле Бегемота.
5. После фильтрации осталось **145 пар** в файле `dataset_behemot_raw.json`.

**Разбиение**: 130 train / 15 val

**Формат**: ChatML

## Модель

**Qwen3-0.6B** (`Qwen/Qwen3-0.6B`) — 596M параметров.

Выбрана потому что:
- Поддержка русского языка и кириллицы в токенизаторе.
- Маленький размер
- Поддержка ChatML-формата промптов.

## Методы дообучения

3 метода SFT (все — 10 эпох):

| Параметр | QLoRA | LoRA (fp16) | Full FT |
|---|---|---|---|
| Обуч. параметры | 10M (2.6%) | 10M (1.7%) | 596M (100%) |
| Квантизация | 4-bit (nf4) | нет | нет |
| Learning rate | 2e-4 | 2e-4 | 5e-5 |
| Batch size | 2 | 2 | 2 |
| Grad. accumulation | 4 | 4 | 4 |
| Scheduler | cosine | cosine | cosine |
| Warmup | 10% | 10% | 10% |
| LoRA r / alpha | 16 / 32 | 16 / 32 | — |
| LoRA target | all-linear | all-linear | — |
| Gradient checkpointing | нет | нет | да |
| VRAM пик | 5.66 GB | 5.97 GB | 13.75 GB |
| Время обучения | 3.8 мин | 2.9 мин | 1.9 мин |

## Результаты

### Метрики стилизации (на валидационной выборке)

| Метод | ROUGE-1 | ROUGE-L | BERTScore F1 | Eval Loss |
|---|---|---|---|---|
| Базовая (без дообучения) | 0.13 | 0.10 | 0.46 | — |
| QLoRA | 0.66 | 0.66 | 0.79 | 4.68 |
| LoRA | 0.64 | 0.64 | 0.76 | 4.74 |
| Full FT | 0.94 | 0.94 | 0.96 | 5.33 |

### Выводы

- Все дообученные модели значительно улучшают стилизацию по сравнению с базовой.
- Full FT показывает наивысшие ROUGE/BERTScore, но при наихудшем eval loss — признак переобучения и запоминания обучающих данных (некоторые ответы дословно совпадают с датасетом).
- QLoRA и LoRA дают сопоставимые результаты при 40–60× меньшем числе обучаемых параметров и 2.4× меньшем потреблении VRAM.
- **QLoRA** — лучший баланс качества и эффективности: чуть лучше метрики, чем LoRA, при наименьшем eval loss.
- Ограничение модели: Qwen3-0.6B иногда вставляет иностранные слова (английский, китайский) из-за мультиязычного токенизатора.
